CharacterStream

public final class CharacterStream : Sequence, IteratorProtocol

Represents a buffered stream of Characters backed by an InputStream.

Implemented as a wrapper on `InputStream`.
The chief advantage here over a regular `InputStream` is that we can peek()
and we get a little more Swift-friendly Characters rather than bytes.
  • init(_:)

    Parameter InputStream: an unopened InputStream. Normally a file but this Swift abstraction allows us to use String-backed InputStreams for ease of testing. The InputStream will be opened and closed by this class according to the RAII idiom.
